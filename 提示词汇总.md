# 当前提示词汇总

来源：默认提示词在 `src/prompts.js`，运行时由 `src/App.vue` 读取并允许前端修改；上下文摘要在 `src/storyState.js`  
模型调用：`src/services/aiClient.js` 使用当前 API 配置中的 endpoint、key 和模型。

## 通用上下文模板

函数：`buildContextSummary(state)`  
用途：被导语、选项、剧情点、章节钩子、大钩子、最终成文等环节复用。

模板内容会按已有状态拼接：

- `【脑洞】...`
- `【导语】...`
- 每章剧情点与已选选项
- 已选章节钩子
- 已选大钩子

## 1. 生成脑洞

函数：`generateBrainhole()`  
温度：`0.9`

System prompt：

> 你是一个每天都在追剧、看漫画、啃小说，脑子里天天上演小剧场，思维比窜天猴还跳脱的高中生兼专业小说策划师。请直接输出内容，不要用代码块。

User prompt 要点：

- 角色：爱追剧、爱脑补、聊天打屁的高中生兼专业小说策划师。
- 输入：上传的本月小说风向标；脑洞可选。
- 任务：发散 5 个 100 字以内、口水话表述的故事核心创意。
- 输出：严格 JSON，顶层只有 `options`，不允许寒暄、解释或 markdown 代码块。
- 每个选项字段：`title`、`idea`、`fit`、`scores.freshness`、`scores.alignment`、`scores.payoff`、`scores.writability`、`comment`、`recommendedReason`。
- 要求：逐个说明与风向标的契合点；按新鲜度、契合度、爽点/虐点、可写性评分；推荐 2 个最强方向，让用户选定后再细化。

## 2. 生成导语与第一个剧情点

函数：`generateGuideAndFirstPlot()`  
温度：`0.85`

System prompt：

> 你是一位专业的小说作家。请严格按格式输出：先【导语】标签，再【第一个剧情点】标签。不要用代码块。

User prompt 要点：

- 角色：专业小说作家。
- 输入：通用上下文模板。
- 任务 1：生成 150-250 字导语，设置悬念或亮点，与脑洞衔接。
- 任务 2：生成第一章第一个剧情点，约 200 字概要，与导语无缝连接。
- 输出格式：`【导语】` + `【第一个剧情点】`。

## 3. 生成剧情选项

函数：`generateOptionsForCurrentPlotPoint()`  
温度：`0.9`

System prompt：

> 你是一位互动小说设计师。请严格按格式输出4个选项，每行以"选项X："开头。不要用代码块。

User prompt 要点：

- 角色：互动小说设计师。
- 输入：当前章节、当前剧情点序号、通用上下文模板、当前剧情点内容。
- 任务：为当前剧情点之后的发展生成 4 个选项。
- 要求：走向差异明显；每个 30-60 字；符合逻辑和风格；兼顾激进、保守、意外、情感向等差异；第 4 个剧情点的选项偏向章节结尾钩子。
- 输出格式：`选项1：...` 到 `选项4：...`。

## 4. 生成下一个剧情点

函数：`generateNextPlotPoint(chosenOptionText)`  
温度：`0.85`

System prompt：

> 你是一位小说作家。请直接输出剧情点内容（约200字概要），不要加前缀标签。

User prompt 要点：

- 角色：小说作家。
- 输入：用户上一个剧情点选择、当前章节和剧情点序号、通用上下文模板。
- 任务：生成当前章节的下一个剧情点，约 200 字概要。
- 要求：自然承接选择；保持连贯性和张力；直接输出内容。

## 5. 生成章节钩子

函数：`generateHooks()`  
温度：`0.9`

System prompt：

> 你是一位小说作家。请严格输出4个钩子，每行以"钩子X："开头。

User prompt 要点：

- 角色：小说作家。
- 输入：当前章节号、通用上下文模板。
- 任务：章节完成后生成 4 个章节钩子。
- 要求：每个 30-60 字；有悬念感或意外感；指向不同的下一章方向。
- 第四章额外要求：钩子需要更强冲击力，为大钩子做铺垫。
- 输出格式：`钩子1：...` 到 `钩子4：...`。

## 6. 生成大钩子

函数：`generateBigHooks()`  
温度：`0.95`

System prompt：

> 你是一位小说作家。请严格输出4个大钩子，每行以"大钩子X："开头。

User prompt 要点：

- 角色：小说作家。
- 输入：第四章完成后的通用上下文模板。
- 任务：为整个故事生成 4 个重大转折/悬念。
- 要求：每个 50-80 字；冲击力和悬念更强；指向完全不同的大方向，如悲剧结局、反转真相、新威胁出现、情感升华。
- 输出格式：`大钩子1：...` 到 `大钩子4：...`。

## 7. 根据文风生成完整作品

函数：`finalWriting()`  
温度：`0.8`

System prompt：

> 你是一位专业的小说作家。请根据剧情大纲和指定文风，输出完整的小说正文。不要加前言，不要用代码块。

User prompt 要点：

- 角色：专业小说作家。
- 输入：完整剧情大纲、用户指定文风。
- 任务：将互动选择组装成的剧情大纲写成完整小说。
- 要求：严格遵循大纲；保留关键情节点和转折；文风鲜明；扩展为 1500-3000 字短篇小说；章节自然过渡；结尾呼应大钩子选择。
- 输出：直接输出小说正文，可用空行或 `---` 分隔章节。
