# 网文AI工业化批量创作系统 PRD

**版本**：V1.0
**目标**：将短篇网文（世情、追妻、脑洞等赛道）的创作过程，拆分为固化的**骨架**与可复用的**技巧/素材**，通过AI实现从脑洞到成文的高效、批量化、低人工参与的工业化流水线。

---

## 1. 产品概述

### 1.1 核心理念
**骨架恒定，要素流动，风格注入。**
-   **骨架**：每个赛道（世情、追妻等）的故事节奏、起承转合节点、冲突爽点密度等结构，是固定的“施工图”。
-   **要素**：灵感选题、人设、具体情节桥段、核心价值观等，是从“素材库”中动态组合与生成的变量。
-   **风格**：基于市场验证的知名作者文风进行封装，为生成的内容统一“外衣”，去AI感。

### 1.2 用户角色
-   **主编/创作者**：定义赛道骨架、审核脑洞与成文、维护知识库质量。
-   **AI系统**：执行拆解、融合、生成、调用、校验等自动化任务。

---

## 2. 核心创作闭环流程

整个系统遵循以下闭环流水线，每一环节都有明确的输入与输出。

1.  **【赛道定义】** → 2. **【脑洞生成】** → 3. **【立项筛选】** → 4. **【骨架匹配】** → 5. **【要素填充 & 细纲生成】** → 6. **【正文生成】** → 7. **【校验与发布】**

-   **流程图**：
    `选定赛道（如追妻文）` → `定义该赛道固定骨架` → `AI融合变量库+风向标生成N个脑洞` → `人工/规则筛选脑洞` → `AI根据脑洞生成人设、核心价值观` → `AI套用骨架，调用节奏锚点和技巧库，生成细纲` → `AI注入指定文风，扩写正文` → `AI校验与人工修改` → `入库`

---

## 3. 详细功能模块

### 3.1 知识库工程（基础设施）

这是整个系统运作的燃料和地基，目标是**将爆款文章拆解为可被AI调用的结构化数据**。

#### 3.1.1 例文拆解器
-   **输入**：F盘/知识库中的爆款例文（按赛道分类：世情20篇、追妻20篇等）。
-   **任务**：使用“D.A.拆解”模块，将每篇例文拆解为以下结构，并以**JSON格式**输出，存入对应子目录。
-   **拆解维度与子目录映射**：
    -   `节奏锚点库`：起承转合各阶段冲突/爽点/悬念/钩子的**具体数量和位置**（如：第1章500字内必须有1个强冲突+1个悬念钩子）。
    -   `情绪地图库`：章节层级的情绪曲线（愤怒→压抑→爽感释放）。
    -   `人物弧光库`：主角成长轨迹、核心欲望与行动力曲线。
    -   `高燃场面库`：结构化的爽点场景模型（如：身份揭晓、打脸、追妻火葬场）。
    -   `故事框架库`：通用大纲模板（如：错嫁流、认亲流、逆袭流的起承转合）。
    -   `人设库`：经典人设标签组合（如：坚韧小白花+霸总，心机女+婆媳）。
    -   `技巧标签库`：具体的写作技法（如：反常规开篇、强悬念切入、对话体交锋）。

#### 3.1.2 变量库（核心融合原料）
-   **定义**：将上述所有拆解出来的子目录内容，全部汇集在一个统一调用逻辑下，称为“变量库”。
-   **作用**：作为生成脑洞、情节和细纲时的“素材与规则池”。
-   **数据格式**：必须采用**JSON**等结构化格式，以支持高效标签调用，避免纯文本带来的“噪音”和效率低下。

#### 3.1.3 文风库（外衣封装器）
-   **来源**：搜集并存储一批经过市场验证、风格固定的知名作者作品。
-   **拆解内容**：不拆情节，只拆解文笔特征。
    -   **句式特点**：长短句偏好、特殊句型。
    -   **分段习惯**：手机阅读友好型分段密度。
    -   **描写手法**：对白与心理描写的比例、转场方式。
-   **调用规则**：一个赛道可绑定一种默认文风，或在生成时指定。

---

### 3.2 赛道骨架工厂（施工蓝图）

这是故事不跑偏的保证，**需要人工与AI共同定义**。

-   **目标**：为每个赛道（追妻、世情、脑洞等）制定一份“创作宪法”。
-   **定义内容**：
    1.  **总纲节奏**：故事必须经过的起承转合4大阶段，以及总章数范围。
    2.  **章节锚点**：以追妻文为例：
        -   **起部分（1-3章）**：每章≥3个冲突点，开篇500字内必须是“反常规”或“强悬念”，章尾必须有强钩子。
        -   **承部分（4-6章）**：每章保持2个冲突，引入1个新悬念，人物开始展现弧光。
        -   **转部分（7-9章）**：核心高燃场面爆发点，情绪到达顶峰。
        -   **合部分（10-12章）**：悬念揭示，核心价值观升华。
-   **存储**：以结构化配置的形式存在，供AI在细纲生成阶段直接套用。

---

### 3.3 脑洞融合与筛选中心（创意引擎）

-   **输入**：`变量库`（全部拆解素材） + `市场风向标`（最新热点、榜单标签）。
-   **处理**：AI执行“融合生成”任务，不进行深度逻辑推理（以节省用量）。它结合变量库中的技巧、人设、高燃场面等资产，融合市场风向标，批量产生脑洞。
-   **输出**：每次生成10个脑洞，每个脑洞应包含：`故事梗概` + `核心冲突` + `底层核心价值观`。
-   **筛选工具**：运用优化后的“**证据链提示词**”和“**立项卡提示词**”，辅助人工快速判断脑洞的市场潜力，做出最终选择。

---

### 3.4 细纲驱动器（核心组装线）

此模块将骨架和脑洞结合，生成详细的施工图纸。包含两个子生成器：

1.  **情节驱动器**：
    -   **输入**：选定脑洞 + 对应赛道骨架的`节奏锚点`。
    -   **任务**：随机/依规则生成具体情节。它根据骨架要求，在“第3章需要一个强冲突”的位置，从`变量库`中调用合适的冲突桥段模型，填充进去。
    -   **输出**：一个体现节奏的地图式情节样本，写明在每个锚点位置发生什么具体事情。

2.  **人物弧光生成器**：
    -   **输入**：脑洞中人设 + 对应赛道的`人物弧光库`。
    -   **任务**：生成主角完整的心理成长路径，并与情节驱动器生成的情节进行咬合。
-   **最终产物**：**细纲**。这是一份融合了情节节奏、人物情绪变化、冲突爽点具体分布的章节规划表。

---

### 3.5 正文生成与校验

-   **组装式生成**：AI根据细纲，逐章进行正文生成。
    -   **关键动作：注入文风**。在生成指令中，强制调用该赛道绑定的`文风库`特征（如“用作家XX的句式和平铺直叙的对话风格，改写下面情节”）。
-   **校验闭环**：生成正文后，进入校验环节。
    -   **AI自动检查**：是否偏离了预设的骨架节奏、核心价值观是否有体现、是否有逻辑漏洞。
    -   **人工复核**：针对开篇、导语、书名以及AI检测出的风险点进行最终修改。
    -   **记录并反哺**：将修改记录和相关数据反馈回系统，用于优化知识库和生成规则，形成闭环进化。

## 4. 非功能性需求与关键问题解决方案

### 4.1 用量成本优化策略
这是系统能否批量运行的关键。

-   **核心原则**：**避免调用AI的深度逻辑推理和长篇分析能力**。
-   **具体措施**：
    -   **拆解与生成分离**：复杂的拆解任务可以离线使用分析能力更强的模型完成，存为结构化数据。
    -   **生成环节“直出”化**：脑洞融合、细纲生成、正文扩写，都采用“根据规则组装”而非“分析后创作”的模式，减少推理Token消耗。
    -   **骨架固化**：最耗能的“布局”工作已由人工预设的骨架完成，AI只负责填空。
    -   **精简提示词**：用结构化JSON传递指令，避免长篇大论的对话式Prompt。

### 4.2 原创性保障（反抄袭机制）
-   **骨架抄袭，内容创造**：只模仿爆款的情绪起伏节奏和冲突频率，这是行业通用结构，不构成抄袭。
-   **脑洞原创**：脑洞是“变量库”中多个元素融合而成，本身就是一次再创造。
-   **人设原创**：人设由脑洞派生，而非对标某一本书。
-   **文风迁移**：模仿的是文笔风格，而非内容，属于合理的风格学习。
